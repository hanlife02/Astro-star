# 前言

如图所示，这是我非常稀疏的网络，也是我目前非常想充实但还非常贫瘠的第二大脑。

上图来自 [graph view](https://obsidian.md/help/plugins/graph)，根据我的双向链接生成的网络图，传统文件夹的线性不符合大脑思考的方式，关联不可能是简单的线性。

粉色的节点代表我的所有内容，蓝色的节点代表我的所有标签。

我的愿景是，数字化我的思考、成果、经验等，借助 AI ，分析和提升自我。

# 我的做法

## 搭建 network

### 我为什么用 Obsidian

我使用 [Obsidian](https://obsidian.md/)，一款非常优秀的笔记软件，拥有丰富的插件、离线的文档和高度可定制化的服务，以及多端同步的功能(关于 sync 我使用 icloud，给🍎加一分)。

### 实现

#### 文件结构

![Obsidian 文件结构](/figures/obsidian-folder.png)

- 1 - Templates ---- 我的内容从此启动

包含 Article,Blog,Daily,Document,Paper 五个模板，分别对应普通文章、博文、日报总结、项目文档、论文。

我做了一些特别的功能设计，例如自动填充属性里的文件创建日期以及更新日期，自动化标签列表等等。这些自动化设计会让我省下更多心思，专注于文章内容，而非如何构建网络。

- 2 - Tags ---- 清晰的标签作为关键节点

我同样做了很多设计，例如当新文章的属性 `tags` 确定或旧文章的 `tags` 修改，都会自动修改对应的标签列表，例如下方 `docs` 标签的示例。这意味着当我创建新内容的时候，只需要简单打上一个 `tag` ，就可以自动加入网络，这同样帮助我专注于内容本身。

![Tags-docs](/figures/Tags-docs.png)

- 3 - Indexs ---- 一切的索引

包含 `Articles` 和 `Tags` 两个部分，分别索引所有的文章和标签，并按一定顺序排序，提供一个非常便捷的地图显示。但我日常启动文件则选择快捷键+搜索启动，需要 Indexs 的原因是为了更好与 AI 交互。

- 4 - Ethan ---- 我真正的第二大脑

这里只有文章，从 Template 启动，被 Tags 标记，在 Indexs 索引，注入我的思考。

前面三个步骤都是自动的，只有思考是最重要的。

- 5 - Attachment ---- 快速的附件存储

文章避免不了附件，例如图片等，所以我做了这个快速存放目录，我在文章里粘贴附件后，便会自动在它下存储，建立双向链接。

- data ---- 存放一些 `json` 格式的数据

例如一些聊天数据，Token 消耗数，以及一些项目需要的数据。

- Token Usage Dashboard ---- 我没有归类，因为这是 Token 消耗面板，我希望尽可能快的打开查看

#### 其他设置

还有一些其他的设置，例如说如何将属性的 tags 做成双向链接，如何在保存后执行更新日期并且执行 Tags 和 Indexs 的更新等。在 `Shell commands` 这个插件的帮助下，我用 AI 写了很多简单脚本解决。

> 没有在这里介绍具体的做法，是因为我觉得因人而异，也许我的习惯、喜好、需求并不是你所需要的，但你依然可以在 agent 的帮助下快速建立起你的 workflow

## 应用 network

我还做了一些其他的服务，帮助我在 network 里更快的填充内容。因为我希望以尽可能快捷的方式，每天让 AI 读取我的尽可能多的思考和总结。另外，实验的记录、生活的感触、严谨的文档，我也都在逐步的完善。

## 关于 obsidian 插件

我只用自己觉得能优化效率的插件，不希望太复杂。如前文所述，我用到了 `Shell commands`,除此之外还有 `Dataview`、`Claudian`、`Easy Typing`、`Linter`、`Advanced Tables`、`Better Word Count ` 等插件，都是帮助我搭建 network 的重要工具。
